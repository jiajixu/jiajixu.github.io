File类

位于java.io.File，用于表示文件和目录

File类只用于表示文件或目录的信息，不能用于对文件内容的访问。

RamdomAccessFile

介绍

支持随机访问文件，可以访问文件的任意位置

  1. java文件模型
    • 在硬盘上的文件是byte存储，是数据的集合
  2. 打开文件
    • 有两种模式“rw”（读写）、“r”（只读）
    • RandomAccessFile raf = new RandomAccessFile(file, "rw");
    • 文件指针：在打开文件时指针在开头 pointer = 0;
  3. 写
    • raf.write(int)
      • 只写一个字节（后8位），同时指正指向下一个位置，准备再次写
  4. 读
    • raf.read()
      • 读一个字节
  5. 文件读写完成后一定要关闭

字节流

InputStream、OutputStream

  • InputStream为应用程序读取数据的抽象
  • OutputStream为应用程序 写入数据的抽象
  • 读到-1就读到了结尾
  • 输入流基本方法
    • int b = in.read();读取一个字节无符号填充到int低八位。-1结束
    • in.read(byte[] buf)读取数据填充到字节数组
    • in.read(byte[] buf, int start, int size)
  • 输出流基本方法
    • out.write(int b)写出一个byte到流，b的低8位。
    • out.write(byte[] buf)将buf字节数组都写入到流
    • out.wirte(byte[] buf, int start, int size)
  • FileInputStream实现了在文件上读取byte数据
  • FileOutputStream实现了向文件中写出byte数据
    • out.write('A')写出’A’的低八位
    • 写入字符：byte[] bs = "阿布".getBytes("gbk"); out.write(bs);
    • write只能写入八位，那么写一个int需要四次

字符流

文本和文本文件

  • java的文本（char）是16位无符号整数，是字符的unicode编码（双字节编码）
  • 文件是byte byte byte …的字节序列
  • 文本文件是文本（char）序列按照某种编码方案（utf-8,utf-16be,gbk）序列化为byte的存储结果

字符流：Reader、Writer

  • 字符的处理，一次处理一个字符
  • 字符的底层任然是基本的字节序列
  • 字符流的基本实现
    • InputStreamReader 根据编码完成byte流解析为char流
    • OutputStreamWriter 按照编码处理char流到byte流

对象的序列化、反序列化

  1. 对象序列化是将Object转换成byte序列，反之叫对象的反序列化

  2. 序列化流（ObjectOutputStream）->writeObject；

    反序列化（ObjectInputStream） -> readObject

  3. 序列化接口（Serializable）

    对象必须实现序列胡接口才能进行序列化

    这个接口没有任何方法，只是一个标准

transient修饰的属性不会默认进行序列化，但可以自行序列化

在ArrayList可以看到writeObject和readObject两个方法对数组进行序列化，其作用是做些优化，不序列化无用的对象。

序列化中子类和父类构造函数的调用问题

一个类实现了序列化接口，那么其子类都可以进行序列化


对子类对象进行反序列化操作时，如果父类没有实现序列化接口，那么其父类的构造函数会被显示调用。
