Linux, 字典

简介

Linux中一切皆文件包括硬件也是文件。

字符界面占用的系统资源资源更少，让操作系统更稳定。

Linux不靠扩展名区分文件类型

约定名字：

压缩包：.gz、.bz2、.tar.bz2 、.tgz等

二进制软件包：.rpm

网页文件:.html、.php

脚本文件：.sh

配置文件：.conf

查询目录中的内容：ls

ls [选项] [文件或目录]

选项：

-a 显示所有文件，包括隐藏文件
-l 显示详细信息（简化命令：ll）
-d 查看目录属性
-h 人性化显示文件大小
-i 显示inode（文件id号）

目录处理命令

创建目录：mkdir -p [目录名]

-p 递归创建目录
命令英文原意：make directories

切换所在目录：cd

cd [目录]

命令英文原意：change directory

简化操作

cd ～    进入当前用户的家目录
cd 
cd -    进入上次目录
cd ..   进入上一级目录
cd .    进入当前目录

相对路径：参照当前所在目录，进行查找

如：  ～$cd ../usr/local/src/

绝对路径：从根目录开始指定，一级一级递归查找。在任何目录下，都能进入指定位置

如：～$cd /etc/

查询所在目录位置：pwd

命令英文原意：print working directory

删除空目录：rmdir [目录名]

命令英文原意：remove empty directories

删除目录或文件：rm

rm -rf [文件或目录]

命令英文原意：remove

选项

-r    删除目录 
-f    强制

复制命令：cp

cp [选项] [原文件或目录] [目标目录]

命令英文原意：copy

选项

-r    复制目录
-p    连带文件属性复制
-d    若原文件是连接文件，则复制连接属性
-a    相当于 -pdr

注意

如果目标目录加上文件名字，则为改名复制。如：mv a.txt ./dir/b.txt(将当前目录的a.txt复制到当前dir目录下，并改名为b.txt)
如果复制并且不改变更新日期则用选项：-a

剪切或改名命令：mv

mv [原文件或目录] [目标目录]

命令英文原意：move

注意

命令不同于”cp“区分文件和目录
当在同一个目录下使用”mv“相当于改名，如：mv a b（将当前目录下的a文件名改为b）

常用目录的作用

  • / 根目录
  • /bin 命令保存目录（普通用户就可以读取的命令）
  • /boot 启动目录，启动相关文件
  • /dev 设备文件保存目录
  • /etc 配置文件保存目录
  • /home 普通用户的家目录
  • /lib 系统库保存目录
  • /mnt 系统挂载目录
  • /media 挂载目录
  • /root 超级用户的家目录
  • /tmp 临时目录
  • /sbin 命令保存目录（超级用户才能使用的目录）
  • /proc 直接写入内存的
  • /sys
  • /usr 系统软件资源目录
    • /usr/bin/系统命令（普通用户）
    • /usr/sbin/系统命令（超级用户）
  • /var系统相关文档内容

连接命令：ln

ln -s [原文件] [目标文件]

命令英文原意：link
功能描述：生成链接文件

选项

-s    创建软链接

注意

如果不在当前目录，软链接必须要写原文件绝对路径

硬链接特征

1. 拥有相同的i节点和存储block块，可以看做是同一个文件
2. 可以通过i节点识别
3. 不能跨分区
4. 不能针对目录使用

软链接特征

1. 类似Windows的快捷方式
2. 软链接拥有自己的i节点和block快，但是数据块中只保存原文件的文件名和i节点号，并没有实际的文件数据。
3. lrwxrwxrwx     l:软链接        软链接文件权限都为rwxrwxrwx
4. 修改任意文件，另一个都改变。
5. 删除原文件，软链接不能使用。

locate [文件名]

-m    只查找帮助文件

搜索命令所在路径及别名

文件搜索命令：find

find [搜索范围] [搜索条件] 

如：
    find /home -name abc
    find /home -mtime +10

搜索条件

-iname  不区分大小写
-user   按照所有者搜索
-nouser 查找没有所有者的文件
-mtime +10 查找10天前修改的文件（[+10：十天前][10：10天当天][-10:10天内]）
-ctime +10 查找10天前改变文件属性的文件
-mtime +10 查找10天前修改文件内容的文件
-size 25k 大小是25KB的文件（[-25k：小于25KB的文件][25:25KB的文件][+25KB：大于25KB的文件][注意：KB单位用小写“k”表示，MB单位用大写“M”表示]）
-inum 6587398 查找i节点是6587398的文件

注意

避免大范围搜索，会非常耗费系统资源
find是在系统当中搜索符合条件的文件名。如果需要匹配，使用通配符匹配，通配符是完全匹配。

Linux中的通配符

*   匹配任意内容
?   匹配任意一个字符
[]  匹配任意一个中括号内的字符
使用，在当前目录查询以a开头的文件：find . -name "a*"

使用条件语句，例子：find /etc -size +20k -a -size -50k

查找/etc/目录下，大于20kb并且小于50kb的文件
-a    and    逻辑与，两个条件都满足
-o    or     逻辑或，两个条件满足一个即可

对搜索结果执行操作，例子：find /etc -size +20k -a -size -50k -exec ls -lh {} \;

查找/etc/目录下，大于20kb并且小于50kb的文件，并显示详细信息
-exec/-ok [命令] {} \;对搜索结果执行操作

搜索字符串命令：grep

grep [选项] 字符串 文件名

在文件中匹配符合条件的字符串

选项

-i    忽略大小写
-v    排除指定字符串

find命令和grep命令的区别

find命令：在系统当中搜索符合条件的文件名，如果需要匹配，时使用通配符匹配，通配符是完全匹配。
grep命令：在文件中搜索符合条件的字符串，如果需要匹配，使用正则表达式进行匹配，正则表达式包含。

查询对应级别的帮助，举例：

查看和命令相关的所有其他命令

“man -k” 命令 相当于 “apropos” 命令

例如：apropos passwd

压缩

常用压缩格式：

.zip
.gz
.bz2
.tar.gz
.tar.bz2

.zip格式压缩

zip [压缩文件名] [源文件]

压缩文件

zip -r [压缩文件名] [源目录]

压缩目录

unzip [压缩文件]

解压缩.zip文件

.gz格式压缩

gzip [源文件]

压缩为.gz格式的压缩文件，源文件会消失

gzip -c [源文件] > [压缩文件]

压缩为.gz格式，源文件保留
例如： gzip -c abc > abc.gz

gzip -r [目录]

压缩目录下所有的子文件，但是不能压缩目录。（每个子文件都被单独压缩）

gzip -d [压缩文件]

解压缩文件

gunzip [压缩文件]

解压缩文件

.bz2格式压缩

bzip2 [源文件]

压缩为.bz2格式，不保留源文件

bzip2 -k [源文件]

压缩之后保留源文件

注意

bzip2命令不能压缩目录

bzip2 -d [压缩文件]

解压缩，-k保留压缩文件

bunzip2 [压缩文件]

解压缩，-k保留压缩文件

打包命令tar

tar -cvf [打包文件名] [源文件]

例如：tar -cvf abc.tar abc

选项

-c：打包
-v：显示过程
-f：指定打包后的文件名

解打包命令

tar -xvf [打包文件名]

例如：tar -xvf abc.tar

选项

-x:接打包

.tar.gz压缩格式

其实.tar.gz格式是先打包成.tar格式，再压缩为.gz格式

tar -zcvf [压缩包名.tar.gz] [源文件]

-z:压缩为.tar.gz格式

tar -zxvf [压缩包名.tar.gz]

-x:解压缩.tar.gz格式

.tar.bz2压缩格式

tar -jcvf [压缩包名.tar.gz] [源文件]

-j:压缩为.tar.bz2格式

tar -jxvf [压缩包名.tar.gz]

-x:解压缩.tar.bz2格式

压缩注意

指定解压缩目录加大C

例如：tar -jxvf dir.tar.bz2 -C /tmp/

可以同时压缩多个文件

例如：tar -zcvf test.tar.gz abc cba

可以指定压缩文件放到哪个目录，只需在压缩文件名前加路径

例如：tar -zcvf /tmp/test.tar.gz abc

只查看压缩包文件，不解压：tar -ztvf [压缩文件]

例如：tar -ztvf test.tar.gz

挂载命令格式

挂载光盘。注：设备文件名：/dev/cdrom是/dev/sr0的软链接。“ -t iso9660”可以不写。

Shell

shell是什么

1. shell是一个命令行解释器，它为用户提供了一个向Linux内核发送请求以便
运行程序的界面系统级程序，用户可以用Shell来启动、挂载、停止甚至是编写一些程序。
2. shell还是一个功能相当强大的编程语言，易编写，易调试，灵活性较强。
shell是解释执行的脚本语言，在shell中可以直接调用Linux系统命令。

输出重定向

符号 作用
[命令] > [文件] 覆盖方式，将正确结果输出到指定文件
[命令] >> [文件] 追加方式，将正确结果输出到指定文件
[命令] 2>[文件] 覆盖方式，将错误命令结果输出到指定文件
[命令] 2>>[文件] 追加方式，将错误命令结果输出到指定文件
[命令] > [文件] 2>&1 覆盖方式，将正确和错误命令结果输出到指定文件
[命令] >> [文件] 2>&1 追加方式，将正确和错误命令结果输出到指定文件
[命令] &>[文件] 覆盖方式，将正确和错误命令结果输出到指定文件
[命令] &>>[文件] 追加方式，将正确和错误命令结果输出到指定文件
[命令] >> [文件1] 2>>[文件2] 追加方式，将正确结果输出到文件1，将错误结果输出到文件2

    0表示键盘设备"/dev/stdin"标准输入
    1表示显示器设备"/dev/sdtout"标准输出
    2表示显示器设备"/dev/sdterr"标准错误输出
不想看到输出过程，如：ls &>/dev/null

wc [选项] [文件名]

-c    统计字节数
-w    统计单词数
-l    统计行数

多命令顺序执行

多命令执行符 格式 作用
; [命令1];[命令2] 多个命令顺序执行，命令之间没有任何逻辑联系
&& [命令1]&&[命令2] 命令1正确则执行命令2，否则不执行命令2
|| [命令1]||[命令2] 命令1不正确才执行命令2，正确则不执行命令2

管道符

[命令1] | [命令2]

命令1的正确输出作为命令2的操作对象
例：
    ls -l /etc/ | more
    netstat -an | grep "ESTABLISHED"

通配符

通配符 作用
? 匹配1个任意字符
* 匹配≥0个任意字符
[] 匹配中括号中任意1个字符。如：[abc]代表匹配或a，或b，或者c
[-] 匹配范围内1个字符。如：[a-z]代表匹配小写字母
[^] 逻辑非，匹配不在括号中的1个字符。如：[^0-9]代表非数字字符

Bash中其他特殊符号

符号 作用
‘’ 单引号。单引号中的所有特殊符号，如”$”、”`“等都没有特殊含义
“” 双引号。双引号中的特殊字符没有特殊含义，除了：”$”、”`“、”\“是例外，
拥有“调用变量的值”、“引用命令”、“转义符”的特殊含义
`` 反引号。反引号扩起来的是系统命令，在Bash中会先执行它
$() 和反引号作用一样，推荐使用这个，不容易看错
# 在Shell脚本中，#开头代表注释
$ 调用变量的值，如：得到name变量的值：$name
\ 转义符，使特殊符变成普通字符
